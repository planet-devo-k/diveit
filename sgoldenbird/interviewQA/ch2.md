#### 리액트에서 Virtual DOM이 무엇인지, 이를 사용하는 이유는 무엇인지 설명해 주세요.

- 실제 DOM의 복사본처럼 React 내부에 존재하는 가상 메모리상의 DOM 구조
- UI가 업데이트될 때 실제 DOM을 바로 조작하지 않고, 먼저 Virtual DOM을 업데이트한 뒤 이전 Virtual DOM과 비교하여 변경된 부분만 실제 DOM에 반영
- 실제 DOM 조작은 변경이 일어날 때마다 reflow와 repaint 같은 무거운 작업이 발생해서 성능에 부담을 줄 수 있고 비용이 큰 작업이기 때문

#### 리액트 생명주기(life cycle)에 대해 설명해 주세요.

- 리액트 컴포넌트는 생성되고, 업데이트되고, 제거되는 과정을 거치는데, 이를 컴포넌트의 생명주기라고 한다.
- 클래스형 컴포넌트에서는 컴포넌트가 화면에 언제 나타나고, 언제 사라지고, 언제 업데이트되는지, 이 과정을 제어할 수 있는 생명주기 메서드들이 제공된다.
- 함수형 컴포넌트에는 명시적인 생명주기 메서드가 없지만 useEffect와 같은 Hook으로 비슷한 패턴을 구현할 수 있다.

#### 메모이제이션을 남용하면 어떤 문제가 생길까요?

- 메모이제이션은 불필요한 연산을 줄여 성능을 개선할 수 있지만, 남용하면 오히려 성능을 저하. 메모이제이션도 비용이 드는 작업이기 때문.

- 값 비교와 저장 자체가 CPU와 메모리를 사용
- 가벼운 연산에까지 메모이제이션을 적용하면 오히려 비효율
- 코드 복잡도 상승, 가독성 저하

#### React Fiber란 무엇인가요?

- React 16부터 도입된 새로운 렌더링 엔진(재조정 알고리즘, Reconciler)
- Fiber는 렌더링 작업을 작은 단위(작업 단위, unit of work)로 쪼개어 중단·재개·우선순위 조정이 가능하게 했다.
- 그 결과 React는 Concurrent Rendering(동시성 렌더링)을 지원할 수 있게 됨.

#### 메모이제이션을 섣불리 사용하는 것은 독이라고 하는 주장과, 렌더링이 비싸니 가능한 모든 곳에서 메모이제이션을 적용해야 한다는 주장이 있습니다. 두 의견 중 어느 쪽에 더 동의하나요? 그리고 그 이유를 설명해주세요.
