# 타입 스크립트란?

자바스크립트의 확장판. 자바스크립트를 더 안전하게 사용할 수 있도록 타입 관련 기능등을 추가한 언어

## 배워야 하는 이유

자바스크립트는 원래 아주 간단한 상호작용 정도를 처리하기 위해 개발되어 아주 유연하게 설계되었는데, 자바스크립트가 node.js가 등장하면서 어디든지 사용될 수 있게 되면서 문제가 발생함 ⇒ 엄격하지 않아 버그 발생 가능성이 높음

더 안정적으로 쓰기위해 타입스크립트가 만들어짐.

## 자바스크립트의 한계점과 타입스크립트

모든 프로그래밍 언어는 타입 시스템을 가지고 있음.

- 동적 타입 시스템
  - 코드를 실행하고 나서 그때그때마다 유동적으로 변수의 타입을 결정
  - 오류가 발생하면 복잡한 프로그램을 만들 경우 오류들이 실행과 동시에 발생하지 않아서 프로그램이 강제 종료되고 서비스가 마비될 수 있음.
  - JavaScript
- 정적 타입 시스템
  - 코드 실행 이전에 타입을 검사하며 모든 변수의 타입을 고정적으로 결정 ⇒ 모든 변수의 타입을 지정해 주어야 함.
  - C, Java

타입스크립트 = 점진적 타입 시스템(동적 타입 시스템 + 정적 타입 시스템)

코드 실행전에 타입을 검사하고, 모든 변수에 일일이 타입을 명시 하지 않아도 됨

# 타입스크립트의 동작 원리

## 대다수의 프로그래밍 언어가 동작하는 방법

대부분의 프로그래밍 언어는 사람 친화적. 그래서 프로그래밍 언어를 컴퓨터가 실행하기 위해 해석하기 쉬운 형태로 변환해야함(컴파일). 컴파일 과정을 거치면 기계어로 변환됨. 이렇게 컴파일 하는 것을 컴파일러라고 불림.

![image.png](attachment:f33b61d0-6e37-45f7-a23a-d44a4696dd1b:image.png)

- 컴파일 과정
  1. 자바스크립트를 AST(추상문법트리) 로 변환
  2. AST를 바이트 코드로 변환

## 타입스크립트의 실행 과정

1. TypeScript를 AST로 변환
2. AST를 보고 타입 검사를 진행 ⇒ 타입오류가 있다면 컴파일 중단
3. AST를 JavaScript로 변환

![image.png](attachment:34556778-5e06-430c-9346-0f0a1d837ed6:image.png)

타입스크립트에 작성한 타입 관련 코드들은 자바스크립트 코드로 변환될 때 사라짐, 프로그램 실행에 영향을 미치지 않음

## 프로젝트 시작 시 해야할 것

1. Node.js 패키지 초기화
2. types node 설치하기(타입 정보)
3. 타입스크립트 컴파일러 설치하기

### 컴파일 명령어

```jsx
tsc [컴파일할 파일]
```

- tsnode - ts 컴파일러 + node

# 타입스크립트 컴파일러 옵션 설정하기

타입스크립트 컴파일러 옵션은 프로젝트마다 설정할 수 있음.

```jsx
tsc --init
```

## tsconfig.json 파일 옵션 설정하기

- Include : tsc가 컴파일할 타입스크립트 파일들의 범위와 위치를 알려줌
  특정 폴더 아래에 있는 모든 타스 파일을 동시에 컴파일 하도록 설정할 수 있음.
- target: 컴파일해서 만들어지는 javaScript의 버전을 설정
- module: 변환되는 자바스크립트의 모듈 시스템을 결정
  CommonJS, ESModule,…
- outdir: 컴파일 결과로 생성될 자바스크립트 파일의 위치 설정
- strict: 컴파일러가 타입 검사 시 얼마나 엄격하게 할 것인가 결정
- module detection
  - 모듈 시스템을 사용하는 문법 키워드를 파일 내에 한 번이라도 작성하면 이 파일을 독립된 모듈로 봄.
- type: 노드가 es 모듈 시스템을 이해할 수 있게 함
