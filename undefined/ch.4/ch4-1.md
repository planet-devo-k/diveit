# Interview

**SPA와 MPA의 차이**

페이지 이동 시 서버로부터 전체 HTML을 다시 받아오는지.

* SPA는 하나의 HTML 위에서 JavaScript가 필요한 부분만 교체하는 식으로 페이지 전환
* MPA는 이동할 때마다 서버에서 새 HTML을 받아와 전체 리렌더링

**웹 페이지 렌더링 방식 CSR, SSR, SSG 각각의 특징과 각 방식을 어떤 상황에 사용하면 좋을지 설명해 주세요.**

HTML이 언제, 어디서 완성되는가

CSR

* 서버는 빈 HTML과 JS 파일만 보냅니다. 브라우저가 JS를 실행해 화면을 직접 그립니다.
* 사용자의 인터랙션이 많고, 개인화된 데이터가 주를 이루는 B2B 대시보드나 웹 앱에 적합합니다.
* 단점: 초기 구동 속도가 느리고, 검색 엔진(SEO) 최적화가 어려울 수 있습니다.

SSR

* 사용자가 요청할 때마다 서버에서 데이터를 채워 HTML을 완성해 보냅니다.
* 데이터가 자주 바뀌면서도 SEO가 중요하고, 초기 화면이 빨리 보여야 하는 커뮤니티나 쇼핑몰에 적합합니다.
* 단점: 서버 부하가 높고, 페이지 이동 시 CSR보다 묵직하게 느껴질 수 있습니다.

SSG (Static Site Generation)

* 빌드 타임에 미리 모든 HTML을 생성해 둡니다. 요청 시 이미 완성된 파일을 전달만 합니다.
* 내용이 거의 바뀌지 않는 블로그, 공식 문서, 랜딩 페이지에 가장 적합하며 속도가 압도적으로 빠릅니다.
* 단점: 데이터가 바뀌면 사이트 전체를 다시 빌드해야 합니다.

**서버 상태와 클라이언트 상태의 차이에 대해 설명해 주세요.**

서버 상태 (Server State)

* 정의: 데이터베이스(DB)에 저장되어 있으며, API를 통해 가져와야 하는 외부 데이터입니다.
* 특징: 비동기적으로 가져오며, 내가 화면을 보고 있는 동안 다른 유저에 의해 바뀔 수 있습니다. (Shared Ownership)
* 관리 도구: React Query, SWR 등을 사용하여 캐싱, 리패칭, 로딩/에러 처리를 수행합니다.

클라이언트 상태 (Client State)

* 정의: 브라우저 메모리에만 존재하며, UI의 현재 모습을 결정하는 임시 데이터입니다.
* 특징: 동기적으로 즉시 접근 가능하며, 오직 현재 사용자만이 제어합니다. (Local Ownership)
* 예시: 모달 오픈 여부, 입력 폼의 텍스트, 다크모드 설정 등.
* 관리 도구: useState, useReducer, Context API, Zustand, Recoil 등.

**리액트에서 말하는 동시성(Concurrency)이란 무엇이며, 어떻게 동작하나요?**

**여러 렌더링 작업**을 동시에 관리하고, 우선순위에 따라 작업을 중단, 재개하거나 폐기할 수 있는 능력

파이버(Fiber) 아키텍처

이를 통해 리액트는 무거운 렌더링 도중에도 사용자의 클릭이나 타이핑 같은 긴급한 이벤트가 들어오면, 기존 작업을 잠시 멈추고 중요한 일을 먼저 처리한 뒤 다시 돌아와 작업을 이어갈 수 있습니다.

**a태그와 Link태그의 차이**

a태그

* 페이지를 이동할 때 모든 리소스를 처음부터 다시 받는다
* 잠시 깜박인 후에 페이지 라우팅
* 렌더링이 어디에서 일어났는지 판단하기 위해 console.log를 실행하면, 서버와 클라이언트 각각 동시에 기록. 즉, 서버에서 렌더링을 수행하고 클라이언트에서 hydrate하는 과정에서 한번 더 실행된다.

next/link

* Next.js에서 제공하는 라우팅 컴포넌트
* 모든 리소스가 아닌 이동한 해당 페이지에 필요한 내용만 받는다.(네트워크 탭 확인)
* console.log가 클라이언트에 기록.
* Next.js는 링크태그가 화면에 보이면(또는 보이기 직전) 해당 페이지의 데이터를 미리 불러온다(자동 prefetch).

**Link vs router.push**

Link

* 사용자 클릭 기반 네비게이션 (화면에 링크 UI가 있을 때)
* SPA이동+자동prefetch
* JSX 안에서만 사용 가능
* a태그 역할이라 검색엔진 인식 용이

router.push

* 코드/이벤트 기반 네비게이션 (조건/이벤트 기반 자동 이동이 필요할 때)
* SPA이동+prefetch 없음
* JS 코드 어디서든 실행 가능
* JS 라우팅이라 SEO 영향 없음(JavaScript로 실행되는 이동이기 때문에 검색엔진이 링크로 인식 못함)
