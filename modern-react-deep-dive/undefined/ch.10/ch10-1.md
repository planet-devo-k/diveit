# Interview

* Suspense는 “로딩 컴포넌트”가 아니라 “렌더링 제어 장치”
* lazy와 suspense는 주로 같이 사용되고, 애플리케이션에서 상대적으로 중요하지 않은 컴포넌트를 분할해 초기 렌더링 속도를 향상시키는 데 많은 도움을 준다.

## Suspense가 기존의 isLoading 상태 기반 렌더링과 다른 점은 무엇인가요?

* 기존 방식은 컴포넌트 내부에서 `if (isLoading) return <Loading />`처럼 명시적으로 로딩 상태를 관리
* Suspense를 사용하면 로딩 처리를 부모 컴포넌트로 위임하여 선언적으로 작성

## React.lazy와 Suspense를 결합하여 얻을 수 있는 성능상 이점은 무엇인가요?

코드 분할 → 처음 페이지에 진입했을 때 다운로드해야 할 파일 크기를 줄이고 초기 렌더링 속도(LCP) 개선

## 모든 컴포넌트를 Suspense로 감싸면 좋을까요?

연관된 데이터라면 하나의 Suspense로 묶어 동시에 나타나게 하거나, Skeleton UI를 적절히 배치하는 기획적 고민이 병행되어야 합니다.
