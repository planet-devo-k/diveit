# ID 정리

## useId: "연결을 위한 이름표"

리액트가 컴포넌트마다 부여하는 내부적인 식별자입니다.

- 언제 쓰나?: `<label htmlFor={id}>`와 `<input id={id}>`를 연결할 때 가장 좋습니다.
- 왜 쓰나?: SSR(서버 사이드 렌더링) 환경에서 서버가 만든 ID와 클라이언트가 만든 ID가 다르면 에러가 나는데, 이를 방지해 줍니다.
- 주의: 이 값을 DB에 저장하거나 리스트의 key로 쓰면 안 됩니다. 리액트 버전이나 환경에 따라 값이 바뀔 수 있기 때문입니다.

## useRef: "값을 담아두는 금고"

리액트의 렌더링 사이클과 상관없이 어떤 값을 보존하기 위한 수단입니다.

- 언제 쓰나?: crypto로 만든 ID를 컴포넌트가 리렌더링되어도 바뀌지 않게 고정하고 싶을 때 사용합니다.
- 왜 쓰나?: 일반 변수(let id = ...)는 함수가 다시 실행되면 초기화되지만, useRef에 담아둔 값은 컴포넌트가 사라질 때까지 유지됩니다.

## crypto.randomUUID(): "진짜 고유한 주민번호"

자바스크립트 엔진 자체가 제공하는 강력한 고유 ID 생성기입니다.

- 언제 쓰나?: 새 게시글을 작성하거나, 새로운 유저 객체를 생성할 때처럼 절대 중복되면 안 되는 ID가 필요할 때 씁니다.
- 왜 쓰나?: useId보다 훨씬 길고 복잡한 난수를 생성하므로, 수만 개의 데이터 사이에서도 충돌할 확률이 거의 없습니다.

## 실전 조합 사례

보통 실무에서는 이 기능들을 섞어서 사용하게 됩니다.

```JavaScript
import { useRef, useId } from 'react';

function UserProfile() {
  // 1. HTML 요소 연결용 (리액트가 관리)
  const htmlId = useId();

  // 2. 데이터 식별용 고유 ID 생성 (내가 직접 관리)
  // 처음 마운트될 때만 crypto를 실행해서 고정시킵니다.
  const dataIdRef = useRef(self.crypto.randomUUID());

  return (
    <div>
      <label htmlFor={htmlId}>사용자 ID: </label>
      <input id={htmlId} value={dataIdRef.current} readOnly />
    </div>
  );
}
```

요약하자면

- 화면 UI 연결이 목적이면 → useId
- 데이터의 고유 키(Key)를 만들고 싶으면 → crypto.randomUUID()
- 그 만든 키를 리렌더링 시에도 유지하고 싶으면 → useRef
