# CSS에서 폰트 적용하기

## 직접 보관 중인 폰트 파일 사용하기 (@font-face)

1. 폰트 정의

```css
@font-face {
  font-family: "NanumPenScript"; /* 우리가 지어준 이름 */
  src: url("/react/onebite/public/NanumPenScript-Regular.ttf");
}
```

2. 실제 적용

```css
body * {
  font-family: "NanumPenScript";
}
```

## 웹 폰트 서비스 사용하기 (Google Fonts 등)

구글 폰트에서 원하는 폰트 선택 후, @import 코드 복사

```css
@import url("https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap");

body {
  font-family: "Noto Sans KR", sans-serif;
}
```

# HTML `<link>` 방식 (권장)

- 구글 폰트에서 원하는 폰트 선택 후, `<link>` 코드 복사
- `<head>` 태그 안에 코드 추가 `<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap" rel="stylesheet">`

가장 큰 이유는 속도(병렬 로딩) 때문입니다.

- 병렬 로딩: 브라우저가 HTML을 읽다가 `<link>`를 만나면, 다른 파일을 읽으면서 동시에 폰트나 CSS를 다운로드하기 시작합니다.
- 브라우저 최적화: 브라우저가 페이지의 구조를 파악하자마자 필요한 리소스를 바로 요청할 수 있어 화면이 뜨는 속도가 빠릅니다.

@import는 직렬 로딩(순차 로딩) 방식이라 속도가 상대적으로 느립니다.

- 대기 시간 발생: 브라우저가 일단 HTML을 읽고, 그 안에 연결된 CSS 파일을 다 다운로드한 뒤, 그 CSS 파일 안에 있는 @import를 발견해야 비로소 폰트를 받기 시작합니다.
- 폭포수(Waterfall) 현상: 앞의 작업이 끝나야 다음 작업을 시작하므로, 네트워크 그래프가 계단식으로 길어지면서 페이지 로딩이 지연됩니다.

# 폰트는 public, 이미지는 assets 폴더에 보관

- 이미지를 최적화할게 아니라면 public에 넣어도 상관없지만 vite가 내부적으로 진행하는 이미지 최적화를 사용하려면 src아래 폴더에 넣어야 한다.
  - npm run build 후 dist 폴더에서 이미지 파일이 최적화되어 있는지 확인
  - npm build후 npm run preview로 빌드된 결과를 미리 확인

- 빌드 후 preview로 개발자도구 element 탭에서 이미지 파일이 최적화되어 있는지 확인해보면
  <br/>public/이미지들은 src가 일만적인 주소로 되어있는 반면
  <br/>src/이미지들은 최적화된 데이터 uri로 되어있는 것을 확인할 수 있다.

- 데이터 URI: 이미지 파일을 문자열 형태로 브라우저 메모리에 캐싱하기 위해 사용되는 포맷 → 새로 고침해도 다시 불러오지 않도록 최적화 된다.

- public/이미지들은 새로고침하면 매번 새롭게 불러온다. 즉, 자바스크립트 번들링 과정에 포함되지 않고 브라우저가 파일의 경로(URL)를 통해 서버에 직접 요청(HTTP Request)

- 기본적으로 번들러(Vite, Webpack 등)는 src 폴더 안에 있는 내용들만 추적해서 하나로 뭉치는(Bundling) 작업을 수행

- public/ 이미지는 경로를 통해 불러올수 있다. (import로 불러올 수 없다.)

- 불러와야하는 이미지가 만개, 십만개 등 너무 많다면 public에 보관하는 것이 나을수도 있다.
  - 브라우저 메모리에 전부 캐싱하면 메모리 용량에 과부하

- 이미지를 불러오는 별도의 모듈 파일을 만들면 사용하기 편리.

```js
// src/assets/get-emotion-images.js
import emotion1 from "./emotion1.png";
import emotion2 from "./emotion2.jpg";
import emotion3 from "./emotion3.gif";
import emotion4 from "./emotion4.svg";
import emotion5 from "./emotion5.webp";

export { emotion1, emotion2, emotion3, emotion4, emotion5 };
```

```js
// src/App.js
import {
  emotion1,
  emotion2,
  emotion3,
  emotion4,
  emotion5,
} from "./assets/get-emotion-images";

export function getEmotionImages(emotionId) {
  switch (emotionId) {
    case 1:
      return emotion1;
    case 2:
      return emotion2;
    case 3:
      return emotion3;
    case 4:
      return emotion4;
    case 5:
      return emotion5;
    default:
      return null;
  }
}
```

```jsx
// src/SomeComponent.js
import { getEmotionImages } from "./App";

return <img src={getEmotionImages(1)} />;
```
