# Interview Questions

- useContext의 사용을 주의점과 엮어서 설명하면?: useContext를 함수 내부에 사용할 시 컴포넌트 재활용이 어려워지고 Provider에 의존성을 갖게 되므로 컴포넌트가 작거나 재사용되지 않을만한 컴포넌트에서 사용

- useReducer의 목적: state 값을 변경하는 시나리오를 제한적으로 두고 이에 대한 변경을 빠르게 확인할 수 있게끔 하는 것이 useReducer의 목적으로 state 사용 로직과 관리 로직을 분리할 수 있어 관리의 편의성 향상

- useState 대신 useReducer를 쓰는 경우: 단순히 number나 boolean과 같이 간단한 값은 useState로 충분하지만 경우의 수가 많아져 state 관리가 어려울 때나 여러 개의 state보다 성격이 비슷한 여러 개의 state를 묶어 관리하는 편이 좋을 때

- useState와 useRef의 가장 큰 차이점은 무엇이며, 각각 언제 사용해야 하나요
