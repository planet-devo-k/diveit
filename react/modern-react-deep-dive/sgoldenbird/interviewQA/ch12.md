#### 이미지 최적화

1. 이미지 포맷 최적화 (Format)

- WebP/AVIF 사용: JPEG나 PNG보다 압축률이 뛰어난 차세대 포맷을 우선적으로 사용합니다.
- 벡터 이미지(SVG) 활용: 로고나 아이콘처럼 단순한 그래픽은 용량이 작고 해상도에 영향을 받지 않는 SVG를 사용합니다.

2. 크기 및 해상도 최적화 (Size & Resolution)
   사용자의 디스플레이 환경에 맞는 적절한 크기의 이미지를 제공합니다.

3. 로딩 전략 최적화 (Loading Strategy)

- Lazy Loading
- Priority (LCP 관리)
- 서버 사이드 렌더링(SSR)
- 직접 호스팅 및 동일 출처(Same-origin) 유지
- 애니메이션 절제
- Layout Shift 방지

4. Next.js next/image 컴포넌트를 활용

- 자동 포맷 변환,
- 리사이징,
- 레이지 로딩,
- 그리고 시각적 안정성을 위한 플레이스홀더(Blur)

5. 리소스 성격에 따른 이원화 관리 전략

- 아이콘 및 그래픽 (SVG Stack & Component)
- 대규모 상품 및 콘텐츠 이미지 (Image CDN)

6. 브라우저 파싱 특성을 고려한 이미지 로딩

- `<img>` 및 `<picture>` 사용:
  HTML 파싱 단계에서 프리로드 스캐너가 이미지를 조기에 발견하여 JS 로딩과 병렬로 다운로드할 수 있게 합니다.

- CSS 배경 이미지 지양
- SVG 내부 `<image>` 주의
- Video Poster 활용

#### 폰트 최적화

1. 폰트 파일 용량 최소화

- 차세대 포맷 사용: WOFF2 형식을 최우선으로 사용
- 서브셋(Subset) 폰트 활용

2. 로딩 우선순위 및 렌더링 제어

- Preload 설정
- font-display: optional

3. 레이아웃 흔들림(CLS) 방지

- Size Adjust 속성: 웹 폰트와 기본 폰트의 비율을 맞추는 size-adjust 속성을 활용해, 폰트가 교체될 때 레이아웃이 튀는 현상을 최소화합니다.

4. 네트워크 비용 최적화

- Self-hosting
- Preconnect 활용
- Local 폰트 활용
