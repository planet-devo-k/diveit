이번에 처음으로 Next.js를 공부했다.
Next.js는 렌더링 전략과 서버 개념까지 포함한 프레임워크라는 점에서 새롭게 느껴졌다.
또한 이미지 최적화, SEO 지원, 캐싱 등 프레임워크 차원에서 제공하는 다양한 최적화 기능이 매우 편리하다 생각했다.

#### 사전 렌더링과 렌더링 전략

SSR, SSG, ISR의 차이를 이해하면서 어떤 상황에서 어떤 전략을 선택해야 하는가를 고민하게 되었다.

#### Page Router → App Router

기존 Page Router 구조와 새롭게 도입된 App Router의 차이를 비교하며 학습했다.
레이아웃 구조, 파일 기반 라우팅, 서버 컴포넌트 개념이 특히 인상 깊었다.

#### React Server Component & 서버 액션

클라이언트와 서버의 역할이 명확히 나뉜다는 점이 처음엔 낯설었지만,
서버 액션을 통해 데이터 생성·조회·재검증까지 구현해보며 흐름이 정리되었다.

#### 데이터 페칭과 캐싱 전략

데이터 캐싱, 요청 메모이제이션, 풀 라우트 캐시, 클라이언트 라우터 캐시 등
단순 fetch가 아니라 캐시 전략을 설계하는 사고가 필요하다는 걸 느꼈다.

#### 고급 라우팅 패턴

parallel route, intercepting route 같은 패턴은 구조를 먼저 이해해야 적용할 수 있었다.

공부를 진행하면서 Next.js가 왜 실무에서 많이 사용되는지 이해하게 되었다.
특히 인상 깊었던 점은 다음과 같다.

- 렌더링 전략을 선택하는 사고
- 서버와 클라이언트의 명확한 역할 분리
- 캐싱과 성능 최적화를 기본 전제로 둔 설계

아직 모든 개념이 완전히 익숙한 것은 아니지만
Next.js로 직접 프로젝트를 구현하면서 이번에 학습한 내용을 적용하며 넥스트에 대한 이해와 응용력을 넓히고 싶다.
